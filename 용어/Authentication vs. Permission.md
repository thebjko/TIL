# Authentication과 Permission의 차이
인증(Authentication)과 권한(Permission)은 액세스 제어와 보안 관련된 개념으로서 서로 다른 의미를 가지고 있습니다.

인증(Authentication)은 사용자가 자신이 주장하는 ***신분 또는 신원을 확인하는 과정***을 말합니다. 사용자가 시스템에 접근하려 할 때, 그들은 자신의 신분을 증명해야 합니다. 대표적으로 아이디(ID)와 비밀번호를 입력하여 로그인하는 과정이 인증의 일례입니다. 다른 인증 방법으로는 생체 인식(지문, 홍채 등)이나 보안 토큰(스마트카드, USB 토큰 등)을 사용하는 것이 있습니다. 인증은 사용자가 신원을 확인하는 단계이며, 인증이 완료되면 사용자는 시스템에 접근할 수 있는 권한을 얻게 됩니다.

반면에 권한(Permission)은 ***인증된 사용자가 시스템 또는 애플리케이션 내에서 수행할 수 있는 특정한 작업이나 기능에 대한 권리***를 나타냅니다. 권한은 보안 정책에 따라 사용자에게 할당되며, 일반적으로 역할 기반 권한 모델을 사용하여 관리됩니다. 사용자는 인증된 후에도 특정 기능을 실행하거나 특정 데이터에 접근하기 위해서는 해당 권한을 가지고 있어야 합니다. 예를 들어, 웹 애플리케이션에서 일반 사용자는 콘텐츠를 읽을 수 있지만 쓰기 권한이 없을 수 있습니다. 반면에 관리자는 읽기와 쓰기 권한을 모두 가지고 있을 수 있습니다.

요약하자면, 인증은 사용자의 신분을 확인하는 과정이며, 인증된 사용자는 권한을 얻습니다. 권한은 사용자가 특정 작업이나 기능을 실행하거나 데이터에 접근하기 위해 필요한 권리를 나타냅니다.
