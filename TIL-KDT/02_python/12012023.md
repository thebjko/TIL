# 클래스
- ## 클래스 속성
	- 예: 많은 원들이 가진 하나의 원주율을 저장
- ## 클래스 메소드
	- `@classmethod` 데코레이터를 사용하여 정의
		- 데코레이터: 함수를 꾸밈
	- 호출 시, 첫번째 인자로 클래스(`cls`)가 전달됨
		- Q: `self`와 무엇이 다른지?
		- A: 목적이 다르다. 목적에 따라 인자를 전달.
			- 예를 들어 클래스를 조작하려면 `cls`라는 이름을 가진 변수를 전달
			- 인스턴스를 조작하려면 `self`라는 이름을 가진 변수 전달
- ## 스태틱 메서드
	- 인스턴스나 클래스를 활용하거나 조작하지 않는 경우. 전달되는 인자 없음

> [!Note]  
> 목적에 따라 인스턴스 메서드, 클래스 메서드, 스태틱 메서드로 정의한다: 인스턴스를 조작하는 인스턴스 메서드, 클래스를 조작하는 클래스 메서드, 둘 다 조작하지 않는 스태틱 메서드.
>  > 실용적인 측면에서 클래스랑 인스턴스의 차이가 헷갈린다.

# 파이썬 응용/심화
> 최근 업데이트에서 추가된 기능들, Syntax sugar
```python
# 1: 조건 표현식
value = num if num//2 == 3 else num - 1

# 2: 열거형 객체
enumerate(range(1,100))   # 열거 객체를 반환함

# 3: List Comprehension
[<expression> for <변수> in <iterable>]
[i**3 for i in range(1,4)]   # [1, 8, 27]

# 4: Dictionary Comprehension
{number: number**3 for number in range(1,4)}

```

## Type Annotation
```python

def g(x : int, y : int) -> int:
	return x + y
	
```
- 명시한 타입으로 변환되는 건 아니지만 에디터를 통해 경고(힌트)를 받을 수 있다.
## Positional-Only Parameters
- 함수를 정의 할 때 어 떻게 호출해야 하는지를 함께 지정
```python
def f(a, b, /, c, d, *, e, f):
	print(a, b, c, d, e, f)

```
> - `a`, `b`는 위치만
> - `c`, `d`는 위치 및 키워드 모두
> - `e`, `f`는 키워드만

> **버전이 업데이트 되면서 달라지는 점에 주목하자**
