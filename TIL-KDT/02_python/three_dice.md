[백준 문제](https://www.acmicpc.net/problem/2480)를 풀던 중, 아래와 같은 구문을 발견했다:
```python
*_,a,b,c=sorted(input());print(['1'+b,c][a<b<c]+'000'[a<c:])
```
세 개의 주사위로 점수를 구하는 문제의 숏코드이다. 하나씩 해석해보자.
## `*_,a,b,c`
`sorted(input())`은 사용자에게 입력값을 받아 오름차순으로 정렬하는 코드이다. 그런데 받은 입력값이 `*_,a,b,c`에 할당되고 있다. `a,b,c`는 알겠는데, `*_`는 뭐지?

애스터리스크(`*`)는 [Extended Iterable Unpacking](https://peps.python.org/pep-3132/), 언더바(`_`)는 마지막 값을 저장하거나, 값을 무시하고 싶을때, 또는 변수나 함수명에 특별한 의미를 부여하고 싶을 때, 또는 숫자 리터럴 값의 자릿수 구분을 위한 구분자로써 사용할 때 사용된다. 여기서는 두 번째 용도일 것이라고 추측해본다. 예를 들어,
```python
x, *_, y = 1,2,3,4,5
```
위와 같이 할당해주면, x에는 1이, y에는 5가 저장된다.  

그렇다면 `*_,a,b,c`는 어떤 용도로 사용되는 걸까? 분명 '주사위 세개' 문제에서 입력값은 한 칸 공백으로 구분된 3개의 정수이다. `*_`를 빼고 제출하면 런타임 에러(`ValueError`)가 발생한다. 그리고
```python
print(sorted(input()))
```
실행 및 숫자 세 개(여기서는 2 4 5) 입력시 출력되는 값은 아래와 같다.
```python
[' ', ' ', '2', '4', '5']
```
앞 2개의 값을 버리기 위해 `*_`를 사용한다.

## `print(['1'+b,c][a<b<c]+'000'[a<c:])`
이 문제는 고수들이 보기에 값을 정수로 변환시킬 필요조차 없는 문제인가보다. 4사분면 문제처럼 정수 변환 없이 풀어버렸다. 
- `['1'+b, c]`는 `['14', '5']`를 출력
- `[a<b<c]`는 boolean값인 `True`를 출력
- `+`는 문자열 붙이기
- `'000'[a<c:]`는 `00`을 출력한다: '5'가 '2'보다 ASCII코드 값이 더 크기 때문에
    - `a<c`가 boolean값을 출력하고 `True`는 1, `False`는 0으로 변환되기 때문에...
> `['14', '5']['1']+'00'`이 되어 `'500'`이 출력되는 것이다.
