# N 까지의 합을 구하는 문제.
숏코딩:
```python
n=int(input())
print(n*-~n//2)
```

## 비트 반전 연산자 `~`

[이 글](https://dojang.io/mod/page/view.php?id=1639#:~:text=~%20%EC%97%B0%EC%82%B0%EC%9E%90%EB%8A%94%20%EB%B9%84%ED%8A%B8%20NOT%20%EC%97%B0%EC%82%B0%EC%9E%90,%22%EB%B9%84%ED%8A%B8%20%EB%B0%98%EC%A0%84%22%EC%9D%B4%EB%9D%BC%EA%B3%A0%20%EB%A7%90%ED%95%A9%EB%8B%88%EB%8B%A4.&text=1010%200010%EC%9D%98%20%EA%B0%81%20%EB%B9%84%ED%8A%B8,%2C%20~162%EB%8A%94%2093%EC%9E%85%EB%8B%88%EB%8B%A4.)에 의하면 1010의 이진수 값은 0101이 되어 10이 5가 되는데, 음수는 어떻게 다루는지 모르겠다. 왜 십진수 100이 십진수 -101이 되는거지?

## 나누기가 아니라 몫?
```python
# n = 100 입력
n=int(input())

# 5050.0 을 출력한다
print(n*-~n/2)
```
아마도 소수점 때문에 값이 정확하지 않아서 몫 연산자로 계산한 듯 하다.
